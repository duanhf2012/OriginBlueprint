# Go 蓝图引擎正确性与性能加固设计

## 目标

在不改变 `GraphDocument` 持久化契约和前端绘制逻辑的前提下，修复 Go 蓝图解析与执行器已确认的异步生命周期、无限执行、Schema 资源消耗、循环条件、legacy 扇出、函数别名和边界输入问题，并在正确性稳定后降低每次执行的内存分配。

## 设计

### 异步执行作用域

新增由一次顶层 `Execution` 拥有、函数子图共享的执行作用域。作用域只承载取消状态、dispatcher、定时器调度器、运行预算和取消钩子；每个函数仍拥有独立 `Graph`、端口上下文、局部变量与函数返回状态。父执行结束、取消或实例释放时，作用域使所有嵌套函数中的 Delay/RPC 续点统一失效。

### 执行安全边界

编译阶段拒绝数据依赖环和新格式执行口扇出。运行阶段对执行节点和纯数据节点统一计步，超过默认预算时返回稳定错误，防止异常 legacy 图、直接构造图或结构校验遗漏导致无限递归。结构化 For/While 由节点内部迭代，仍受同一预算约束。

### Schema 限制

对节点端口 ID、端口总数、动态 Sequence 输出数及函数签名端口数设置宽松但有限的常量。重复端口 ID、负数和超过限制的值必须返回带节点/字段上下文的错误，不允许静默覆盖或按外部数值直接分配巨型切片。

### 执行语义

While 在每轮判断前重新求值连接到条件输入的数据生产链。legacy 同一执行输出口的多条边按文档边顺序全部执行；新格式仍拒绝该结构，要求显式 Sequence。加载函数库时，图名、函数 ID 和路径别名冲突直接失败并报告两个来源。

### 边界与并发

固定 Continuation 只允许绑定真实执行输出口。随机整数使用无溢出的闭区间采样。`GraphInstance` 变量保持线程安全的单次读写语义；复合读改写不承诺原子性，并通过文档和测试明确。

### 性能

先用执行级预分配复用节点上下文槽位，避免每次进入节点都重新创建端口切片；同一节点递归或重入时必须获得独立帧，异步 Continuation 持有的帧在恢复完成前不得复用。Execution 完成后释放参数和临时图引用，仅保留结果快照。禁止在生命周期证明完成前引入 `sync.Pool`。

## 验证

每项修复遵循失败测试、最小实现、窄测试、全包测试的顺序。最终运行 `go test ./engine/go/blueprint -count=1`、`go test -race ./engine/go/blueprint -count=1`、`go vet ./engine/go/blueprint`、相关 benchmark，并记录 `go test ./... -count=1` 中任何与本次修改无关的既有失败。
