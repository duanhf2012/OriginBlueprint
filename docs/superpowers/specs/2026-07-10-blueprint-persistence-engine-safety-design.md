# 蓝图持久化与执行引擎安全修复设计

日期：2026-07-10

## 1. 背景

本设计针对蓝图文件往返转换、Go 执行引擎校验、异步生命周期、热加载和函数局部变量语义进行收敛。修复目标是阻止静默数据丢失、释放后继续执行和非法图进入运行时，同时保持现有蓝图绘制、端口显示、节点 JSON 加载和合法流程环路行为不变。

现状审计已经确认：

- 158 个 legacy `.vgf` 样本中有 14 处未映射 `port_defaultv` 在导入后再导出时丢失。
- legacy 导出遇到无法映射的节点、连接或端口时会跳过，而不是报错。
- `Sleep` 的延迟回调在实例释放后仍可继续执行。
- 外部定时器触发后没有从实例的定时器集合中移除。
- 编译器对重复节点 ID、端口越界、执行/数据端口方向错误和内建端口类型不匹配校验不足。
- 热加载只替换编译结果，没有按新定义重建实例变量。
- 函数调用当前会创建新的变量表，这一行为正确；但共享父级变量锁的实现和“函数共享实例变量”的文档描述均不正确。

## 2. 目标

1. legacy `.vgf` 的已知内容必须可往返保存，无法表达的内容必须显式失败，不能静默丢失。
2. 编译阶段拒绝确定无效的图结构，避免运行时端口类型被意外改写。
3. 蓝图实例释放后，属于该实例的异步任务不能恢复执行。
4. 热加载后，新执行使用新图定义和一致的新变量快照；进行中的执行不被中途换底。
5. 函数蓝图中的变量是调用帧局部变量，每次调用、递归调用都独立，且与调用方同名变量隔离。
6. 修改保持局部，不触碰 Rete 绘制、节点视觉、节点 schema 解析和现有保存格式选择策略。

## 3. 非目标

- 本轮不统一 `.obp` 与 `.vgf` 的文件语义。
- 本轮不恢复缺失的业务节点定义。
- 本轮不全面禁止执行环路。合法循环节点依赖回边，循环防护应作为独立设计处理。
- 本轮不允许函数读取或写入调用方蓝图变量。
- 本轮不重构同步执行热路径，也不以减少 `clonePorts` 分配为主要目标。
- 本轮不修改蓝图节点和端口的显示、布局、连线交互。

## 4. Legacy 默认值残留数据

### 4.1 数据结构

在 `GraphLegacyState` 中增加可选残留字段，用可见节点 ID 关联无法映射到当前 schema 输入端口的 legacy 默认值：

```json
{
  "legacy": {
    "format": "vgf",
    "residualNodeDefaults": {
      "node-17": {
        "class": "DoHitEffect",
        "values": {
          "13": 0
        }
      }
    }
  }
}
```

Go 数据契约：

```go
type GraphLegacyResidualDefaults struct {
    Class  string                 `json:"class"`
    Values map[string]interface{} `json:"values"`
}

type GraphLegacyState struct {
    // 现有字段保持不变。
    ResidualNodeDefaults map[string]GraphLegacyResidualDefaults `json:"residualNodeDefaults,omitempty"`
}
```

TypeScript 的 `GraphLegacyState` 增加同构可选字段。前端继续把 `legacy` 当作不透明持久化数据深拷贝，不把残留默认值渲染成额外端口或控件。

### 4.2 导入规则

1. 根据 legacy class 的当前 schema，把可映射输入默认值写入节点可编辑的 `values`。
2. 对索引超出当前 schema 输入数量、但存在于原始 `port_defaultv` 的值，按原索引字符串写入 `residualNodeDefaults[nodeID].values`。
3. 仅当一个节点确实存在残留值时创建对应条目。
4. 残留条目同时记录原始 legacy class，防止节点 ID 被复用或节点类型变化后错误套用旧值。
5. 原始数组中的显式 `null`、`false`、`0` 和空字符串都视为数据，不按真假值过滤。

### 4.3 导出规则

导出某个节点时按以下优先级构建 `port_defaultv`：

1. 若残留条目的 `class` 与当前导出 class 完全一致，先按索引恢复残留值。
2. 再把当前 schema 可映射端口的编辑值覆盖到对应索引。
3. 当前可编辑值优先于残留值。
4. class 不一致时忽略该节点的残留值，不把旧类型数据写入新类型。
5. 数组长度取当前映射索引与有效残留索引的最大值，并保持中间索引位置。

该设计只保存当前编辑器无法表达、但 legacy 文件已有的数据，不扩展可见节点端口，也不把未知字段提升为运行时输入。

## 5. Legacy 导出预检

legacy 导出改为失败关闭。开始构造输出前，完整检查：

- 每个可见节点都能映射到 legacy class 和端口规格。
- 每条可见连接的源节点和目标节点都存在且可映射。
- 每条连接的源、目标端口都能映射到 legacy 端口索引。
- 端口方向与连接类型满足 legacy 表达要求。

任一条件失败时，导出返回包含节点 ID、连接 ID、端口键和失败原因的错误。不得跳过对象后继续生成部分文件，也不得自动改存 native GraphDocument。

预检必须在写文件之前完成。前端保存流程捕获错误，显示可操作的错误消息，并保持标签页为未保存状态。失败不得覆盖已有目标文件。

## 6. 编译器结构校验

### 6.1 Schema 版本

当前引擎只接受 `schemaVersion == 1`。`0`、负数和大于 `1` 的版本都返回明确错误，防止把未知新格式按旧语义执行。

### 6.2 节点与连接

编译阶段拒绝：

- 空节点 ID。
- 重复节点 ID。
- 连接引用不存在的源或目标节点。
- 源输出端口索引越界或目标输入端口索引越界。
- 执行端口连接到数据端口，或数据端口连接到执行端口。
- 两端均为内建 `*Port` 且声明的数据类型不兼容。

对于自定义 `IPort`，编译器不猜测其类型系统；只执行接口层面可确认的方向和存在性校验。自定义端口的额外兼容性由其实现负责。

编译器不新增“图中存在任意环就失败”的规则。连接建立后，运行时也不得通过 `SetValue` 把目标端口的声明类型改成源端口类型；类型不兼容必须在编译阶段失败。

## 7. 异步任务与实例释放

### 7.1 实例释放信号

每个运行实例持有只关闭一次的 release 信号。所有可挂起执行都监听该信号：

- `Sleep` 在定时器到期与 release 信号之间选择。
- release 先发生时停止定时器，并且不调用 continuation。
- continuation 执行前再次确认实例仍有效，覆盖定时器和释放几乎同时发生的竞态。

### 7.2 外部定时器

实例维护自己创建的外部定时器 ID。定时器正常触发时，先在锁内从集合中消费 ID，再解锁并触发后续模块调用。重复回调或释放后的回调不得再次进入蓝图。

释放实例时：

1. 在蓝图实例锁内移除实例并关闭 release 信号。
2. 在定时器锁内取出并清空该实例的定时器 ID。
3. 释放所有内部锁。
4. 在锁外调用外部模块取消定时器。

无外部模块时创建的本地定时器也必须登记到实例生命周期中，确保后续取消和实例释放都能找到同一个定时器，而不是只存在于单次执行会话。

### 7.3 锁顺序与回调边界

- 蓝图实例锁保护实例集合，以及编译结果和变量快照的原子替换。
- 变量锁只保护某一份变量映射的读取、克隆和修改。
- 定时器锁只保护定时器登记表。
- 不在持有上述任一锁时调用外部模块、用户回调或 continuation。
- 需要跨资源操作时先在各自锁内提取不可变工作项，解锁后执行外部动作。

该约束避免外部模块重入蓝图时产生锁反转或死锁。

## 8. 热加载变量快照

热加载采用复制后交换：

1. 编译新 GraphDocument，失败时保留旧编译结果和旧实例状态。
2. 对每个实例构建新的顶层变量映射。
3. 名称和声明类型都相同的变量，克隆保留当前值。
4. 新增变量使用新文档中的默认值。
5. 类型变化的变量重置为新文档默认值。
6. 已删除变量不进入新映射。
7. 新编译结果与新变量映射在实例锁内一次交换。

一次执行会话在开始时捕获编译结果和变量映射快照。已经开始或挂起中的会话继续使用旧快照，新会话使用新快照，禁止在一条执行链中途混用新旧节点表或变量表。

这里的迁移只适用于长期存在的顶层蓝图实例变量，不适用于函数局部变量。

## 9. 函数局部变量语义

函数蓝图中的变量遵循调用帧语义：

- 每次调用都从函数蓝图变量定义和默认值创建全新的局部变量映射。
- 上一次调用对局部变量的修改不能影响下一次调用。
- 递归调用的每一层都有独立局部变量映射。
- 函数局部变量与调用方蓝图中的同名变量互不影响。
- 函数挂起后恢复时继续使用该次调用自己的局部变量映射。
- 函数返回后，该调用帧不再被顶层实例或后续调用引用。

函数子图使用自己的变量锁，不共享调用方实例的变量锁。函数参数和返回值只通过既有入口、出口端口传递，不通过同名变量隐式共享。

维护文档中“函数调用共享实例变量”的描述必须改为上述规则。

## 10. 测试策略

### 10.1 Legacy 往返

- 对仓库中 158 个 legacy `.vgf` 样本执行导入再导出。
- 精确比较节点数、连接数和所有 `port_defaultv`，包括 `null`、`false`、`0` 和空字符串。
- 增加节点 ID 复用但 class 改变时不恢复残留值的测试。
- 增加未知节点、未知端口和不可映射连接导出失败且不生成部分文件的测试。
- 增加前端保存失败后标签仍为 dirty、原文件未被覆盖的测试。

### 10.2 编译器

- 覆盖 schema 版本不支持、空/重复节点 ID、端口越界、方向错误和内建类型不匹配。
- 覆盖合法流程回边仍可编译。
- 验证类型错误不会改变目标端口声明类型。

### 10.3 生命周期

- `Sleep` 到期前释放实例，后续节点不得执行。
- 外部定时器正常触发后从实例集合移除。
- 实例释放取消仍登记的外部和本地定时器。
- 定时器触发与释放并发执行，多次重复运行并通过 race 检查。
- 回调重入蓝图不发生死锁。

### 10.4 热加载

- 同名同类型变量保留当前值。
- 新变量获得默认值，删除变量消失，改类型变量重置。
- 编译失败保持旧状态。
- 挂起中的旧会话使用旧快照，热加载后的新会话使用新快照。

### 10.5 函数局部变量

- 连续两次调用时，第二次从默认局部变量开始。
- 调用方和函数存在同名变量时互不影响。
- 两层递归调用变量隔离。
- 函数经过异步挂起再恢复时保留本调用局部状态。
- 函数返回后没有被顶层实例持有的调用帧引用。

## 11. 验收命令

按由窄到宽的顺序执行：

```powershell
go test ./engine/go/blueprint -count=1
go test -race ./engine/go/blueprint -count=1
go test ./engine/go/blueprint -count=20
go test ./...
go vet ./...
```

前端执行：

```powershell
cd frontend
npm.cmd run test:layout
npm.cmd run build
```

最后重新运行全量 legacy 样本往返审计和已有引擎基准。同步执行基准不得出现明显回退；异步生命周期测试不得出现释放后执行或 race 报告。

当前全量测试中依赖缺失业务 schema 的三个样本测试应单独记录，不通过放宽编译、迁移或导出规则来掩盖。若这些测试在实现前仍失败，报告必须区分既有环境问题与本轮回归。

## 12. 风险与缓解

- **旧文件残留数据污染新节点**：残留值同时绑定节点 ID 和 legacy class，class 变化时不恢复。
- **导出规则变严格影响现有工作流**：错误包含具体对象和原因，保存失败保持 dirty 且不覆盖原文件；不再生成表面成功的残缺文件。
- **锁调整引入死锁**：内部锁只用于提取或交换状态，所有外部调用和 continuation 都在锁外；增加重入和 race 测试。
- **热加载混用新旧状态**：执行会话捕获不可变快照，交换发生在事务边界。
- **函数变量被误并入实例迁移**：顶层实例变量与函数调用帧使用不同创建路径和测试矩阵。
- **严格类型校验拒绝自定义端口**：只对内建 `*Port` 做声明类型比较，自定义 `IPort` 不做推断。
- **合法循环被误判**：本轮不做全图无环校验，只校验连接结构和端口契约。

## 13. 实施顺序

1. 先用失败测试固定 legacy 默认值丢失和导出静默跳过行为，再实现残留数据与预检。
2. 增加编译器失败测试，再补结构和版本校验。
3. 增加释放后恢复、定时器消费和重入测试，再调整异步生命周期。
4. 增加热加载变量迁移与会话快照测试，再实现复制后交换。
5. 增加函数局部变量隔离回归测试，移除父级变量锁共享并修正文档。
6. 运行窄测试、race、重复测试、全量测试、前端测试、样本往返审计和基准对比。

每一步保持可独立验证，不把文件格式、并发和函数语义改动合并成一个不可定位的提交。
